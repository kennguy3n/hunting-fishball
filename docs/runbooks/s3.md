# S3-compatible object storage runbook (`s3`)

> Source: [`internal/connector/s3/s3.go`](../../internal/connector/s3/s3.go)
> · API base: `https://s3.<region>.amazonaws.com` (configurable per source)
> · Webhook: not supported (poll-only)
> · Delta cursor: `start-after` key OR `last-modified` ISO 8601

## 1. Auth model

Static AWS-style credentials (Access Key ID + Secret Access Key)
authenticated with AWS Signature Version 4. Compatible with AWS
S3, MinIO, Cloudflare R2, Backblaze B2 (S3-compat), Wasabi, and
similar S3-API-compatible buckets.

```json
{
  "access_key_id": "AKIA...",
  "secret_access_key": "...",
  "endpoint": "https://s3.us-east-1.amazonaws.com",
  "bucket": "my-bucket",
  "region": "us-east-1"
}
```

SigV4 implementation:
[`internal/connector/s3/sigv4.go`](../../internal/connector/s3/sigv4.go).
The connector signs every request with canonical headers; no
vendor SDK is used.

## 2. Credential rotation

Routine rotation:

1. AWS IAM → **Users** → connector's IAM user → **Security
   credentials** → "Create access key" to mint a fresh key pair.
2. PATCH the source credential via the admin portal.
3. Wait for any in-flight requests to complete (`SELECT count(*)
   FROM source_runs WHERE state='running'`).
4. AWS IAM → **Inactivate** the old key. After 24h of confirmed
   unused state, delete it.

Emergency rotation (key leak):

1. AWS IAM → **Inactivate** the leaked key immediately.
2. Pause the source in the admin portal.
3. Mint a replacement key, patch the credential, re-enable the
   source.
4. Audit CloudTrail for any GetObject/ListBucket calls signed
   with the leaked key between leak and inactivation.

## 3. Quota / rate-limit incidents

AWS S3's per-prefix request rate is 5,500 GET / 3,500 PUT per
second per prefix; bucket-wide limits are much higher but each
prefix is independent. S3-compat backends typically enforce much
lower limits (MinIO defaults to 1000 RPS, R2 to 1000 RPS).

S3 surfaces throttling as:

- `503 Slow Down` (S3-specific).
- `429 Too Many Requests` (some S3-compat backends).

Both wrap to [`connector.ErrRateLimited`](../../internal/connector/source_connector.go);
the adaptive rate limiter halves the per-source RPS on receipt.

Knobs:

- Split the bucket into multiple sources, each scoped to a
  different prefix (`prefix` filter in the connector config) to
  spread the per-prefix budget.
- `CONTEXT_ENGINE_RATELIMIT_<source_id>_RPS` floors the
  per-source RPS; default 100.

## 4. Outage detection & recovery

AWS S3 outages are tracked at https://status.aws.amazon.com.
S3-compat backends each have their own status pages.

Common outage signatures:

- `503 Service Unavailable` or `503 Slow Down` from
  `s3.<region>.amazonaws.com`.
- `InternalError` with no body.
- Bucket policy / SCP change locking out the IAM user (this
  surfaces as `403 AccessDenied`, not an outage).

Recovery:

1. Confirm the outage scope (regional vs. account-specific) via
   the status page and CloudTrail.
2. Once recovered, the iterator resumes from the last successful
   `start-after` key automatically (idempotent listing).
3. For S3-compat backends with no status page (R2, MinIO),
   probe with `aws s3 ls s3://bucket --endpoint-url <endpoint>`.

## 5. Common error codes

| Upstream | Body | Meaning | Action |
|----------|------|---------|--------|
| `403` | `<Error><Code>AccessDenied</Code></Error>` | IAM policy revoked GetObject/ListBucket | Re-grant `s3:GetObject` + `s3:ListBucket` on the bucket ARN |
| `403` | `<Error><Code>SignatureDoesNotMatch</Code></Error>` | Clock skew >15min OR wrong secret | Sync host clock via NTP; verify credentials in admin portal |
| `404` | `<Error><Code>NoSuchBucket</Code></Error>` | Bucket deleted or wrong region | Remove source; recreate with correct region |
| `429` | n/a | Per-bucket request rate exceeded | §3; lower per-source RPS or split prefixes |
| `503` | `<Error><Code>SlowDown</Code></Error>` | Hot-spot on a prefix | §3; rebalance via prefix sharding |
| `503` | n/a | Upstream outage | §4 |
